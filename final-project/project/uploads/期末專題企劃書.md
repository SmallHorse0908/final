## 五、系統功能規劃

### 5.1 系統整體架構

本網站採 Django 之 **MTV（Model–Template–View）** 架構開發，整體運作流程如下：

1. 使用者透過瀏覽器發出請求 → Django **URL Dispatcher** 對應到指定 View。
2. **View（views.py）** 處理測驗邏輯，從 **Model** 讀取題目並呼叫亂數打散工具。
3. **Template（HTML）** 將題目渲染成作答頁面回傳給使用者。
4. 使用者送出答案後，View 比對正解並計算分數，最後將結果頁回傳至前端。

```
[Browser] ⇄ [URLs] ⇄ [Views] ⇄ [Models / Database]
                                ⇩
                            [Templates]
```

### 5.2 主要頁面（前端流程）

| 頁面 | 主要內容 |
| --- | --- |
| 首頁 `/` | 顯示網站簡介、章節列表、開始測驗按鈕 |
| 章節頁 `/quiz/<chapter_id>/` | 顯示該章節題數與「開始作答」按鈕 |
| 作答頁 `/quiz/<chapter_id>/start/` | 隨機出題、隨機排列選項，逐題或整卷作答 |
| 結果頁 `/quiz/result/` | 顯示得分、正確率、每題正解與作答對照 |
| 後台 `/admin/` | 管理者登入後可新增／修改／刪除題目 |

### 5.3 基本功能（必做）

#### (1) 題庫管理（Django Admin）
- 透過內建 Django Admin 後台進行題目維護，免額外撰寫 CRUD 介面。
- 可新增、修改、刪除題目與選項，並設定正確答案。
- 題目可依章節分類，方便依主題管理。
- 管理者帳號透過 `createsuperuser` 建立並登入後台。

#### (2) 隨機出題順序
- 使用 Python 內建 `random.shuffle()` 將題目清單隨機打散。
- 每次進入測驗時皆重新洗牌，避免使用者背題目位置。
- 可於 View 層控制每次測驗抽出的題目數量（例：從題庫隨機抽 10 題）。

#### (3) 隨機選項排列
- 進入作答頁時，將每題之選項清單以亂數排列後再渲染至 Template。
- 即便題目相同，選項位置每次皆不同，避免「死記答案位置」。
- 後端會記錄打散後的選項與原本正解之對應，確保比對正確。

#### (4) 作答後顯示正確答案
- 使用者送出答案後跳轉至結果頁，顯示：
  - 總分與正確題數／總題數。
  - 每一題的「使用者作答」與「正確答案」對照。
  - 答對者以綠色標示、答錯者以紅色標示，視覺上一目了然。

### 5.4 進階功能（規劃中，視時程實作）

| 功能 | 說明 |
| --- | --- |
| 計時功能 | 作答頁面顯示倒數計時器，時間到自動交卷 |
| 作答記錄 | 將每次測驗結果存入資料庫，可於個人頁面查看歷史成績 |
| 排行榜 | 依分數或作答時間排序，顯示 Top 10 |
| 難易度分級 | 題目可標記為簡單／中等／困難，作答前可選擇難度 |
| 錯題複習模式 | 系統自動蒐集使用者答錯的題目，建立專屬複習題組 |
| 章節綜合測驗 | 跨章節隨機抽題，模擬期中／期末考形式 |

### 5.5 資料模型概念

題庫採「章節 → 題目 → 選項」三層式設計：

| Model | 主要欄位 |
| --- | --- |
| `Chapter`（章節） | 章節編號、章節名稱 |
| `Question`（題目） | 題幹、所屬章節、（選做：難易度） |
| `Choice`（選項） | 選項文字、是否為正解、所屬題目 |

### 5.6 UI／UX 設計重點

- **配色簡潔**：以淺色系為主，重點按鈕（提交、下一題）使用對比色。
- **RWD 響應式**：使用 CSS Flexbox／Grid，於桌機與手機皆能正常顯示。
- **作答提示明確**：選項以大區塊呈現，點擊範圍大、易於操作。
- **結果視覺化**：以顏色（綠／紅）與圖示標示對錯，提升回饋感。

## 五、系統功能規劃

### 5.1 系統整體架構

本網站採 Django 之 **MTV（Model–Template–View）** 架構開發，整體運作流程如下：

1. 使用者透過瀏覽器發出請求 → Django **URL Dispatcher** 對應到指定 View。
2. **View（views.py）** 處理測驗邏輯，從 **Model** 讀取題目並呼叫亂數打散工具。
3. **Template（HTML）** 將題目渲染成作答頁面回傳給使用者。
4. 使用者送出答案後，View 比對正解並計算分數，結果存入資料庫，最後將結果頁回傳。

```
[Browser] ⇄ [URLs] ⇄ [Views] ⇄ [Models / Database]
                                ⇩
                            [Templates]
```

### 5.2 主要頁面（前端流程）

| 頁面 | 路由 | 主要內容 |
| --- | --- | --- |
| 登入頁 | `/set-name/` | 輸入使用者名稱（無密碼），存入 Session |
| 首頁 | `/` | 顯示網站簡介、章節列表、個人最佳成績、功能說明 |
| 章節頁 | `/quiz/<chapter_id>/` | 顯示該章節題數與個人歷史成績，提供「開始作答」按鈕 |
| 作答頁 | `/quiz/<chapter_id>/start/` | 隨機出題、即時回饋、15 分鐘倒數計時 |
| 結果頁 | `/quiz/result/` | 顯示得分、等第評語、每題正解與作答對照 |
| 排行榜 | `/leaderboard/` | Top 10 玩家，依分數降序、用時升序排列 |
| 歷史成績 | `/history/` | 個人所有作答紀錄與統計（平均分、最佳分、總用時） |
| 錯題複習 | `/review/` | 自動蒐集仍未答對的題目，顯示解析 |
| 後台管理 | `/admin/` | Django Admin，管理者可進行題庫 CRUD |

### 5.3 基本功能（必做）✅

#### (1) 題庫管理（Django Admin）
- 透過內建 Django Admin 後台進行題目維護。
- 可新增、修改、刪除題目與選項，並設定正確答案。
- 支援程式碼區塊（`block`）與解析說明（`hint`）欄位。
- 題目依章節分類；管理者帳號透過 `createsuperuser` 建立。

#### (2) 隨機出題順序 ✅
- 使用 Python 內建 `random.shuffle()` 將題目清單隨機打散。
- 每次進入測驗時皆重新洗牌，避免使用者背題目位置。

#### (3) 隨機選項排列 ✅
- 進入作答頁時，將每題之選項清單以亂數排列後再渲染至 Template。
- 後端計算時以 `question.id` 對應，與選項順序無關。

#### (4) 作答後顯示正確答案 ✅
- 即時回饋：選取選項後立刻以顏色標示對錯，並顯示解析說明。
- 結果頁顯示每一題的「使用者作答」與「正確答案」對照。

### 5.5 資料模型

題庫採「章節 → 題目 → 選項」三層式設計，並額外有作答記錄：

| Model | 主要欄位 |
| --- | --- |
| `Chapter`（章節） | 章節編號、章節名稱 |
| `Question`（題目） | 題幹、程式碼區塊、解析、難易度、所屬章節 |
| `Choice`（選項） | 選項文字、是否為正解、所屬題目 |
| `QuizRecord`（紀錄） | 使用者名稱、章節、得分、正確題數、用時、每題明細 |

### 5.6 UI／UX 設計重點

- **深色終端機風格**：背景 `#0a0a0a`，以 JetBrains Mono 等寬字型呈現，強化程式感。
- **色彩語意**：綠色（正確）、紅色（錯誤）、琥珀色（警示）、青色（強調）。
- **RWD 響應式**：使用 CSS Flexbox／Grid，桌機與手機皆能正常顯示。
- **即時回饋**：作答後立刻標示對錯，不需等到交卷才知道答案。
- **導覽點設計**：上方小圓點標示作答進度，綠色＝已答對、紅色＝已答錯、白色＝未作答。
